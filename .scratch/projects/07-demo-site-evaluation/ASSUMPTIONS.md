# Demo Site Evaluation - Assumptions

## Purpose
This demo site will serve as a comprehensive evaluation and showcase of bullish-ssg's capabilities, hosted on GitHub Pages. It should demonstrate all implemented features and serve as both documentation and a proving ground.

## Target Audience
- Library users wanting to see features in action
- Contributors evaluating the tool
- Maintainers verifying functionality

## Site Goals
1. **Documentation**: Clear explanation of all bullish-ssg features
2. **Demo**: Live examples of vault linking, wikilinks, validation, etc.
3. **Evaluation**: Comprehensive test coverage of real-world scenarios
4. **Verification**: Proof that the library works end-to-end

## Constraints
- Must be deployable to GitHub Pages
- Should use both direct mode and symlink mode (for testing)
- Must exercise all CLI commands: init, link-vault, build, serve, validate, check-links, deploy
- Should include intentional broken links for validation testing

## Content Structure Ideas
- Getting Started Guide
- Feature Showcase with examples
- Blog posts demonstrating various content types
- Interactive validation examples
- Configuration reference
- Troubleshooting section
